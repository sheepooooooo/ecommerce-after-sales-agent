# 可溯源政策问答 RAG 设计

## 业务目标

第三阶段 C 的目标是在政策检索能力之上，构建一个受证据约束的政策问答服务。它只回答政策咨询，不查询订单、不判断某笔订单退款资格、不执行真实售后动作。

## 数据流

用户问题进入 `policy_qa_tool` 后，服务调用政策检索 Tool 获取 TopK chunk。若无相关政策，则不调用 LLM，直接返回 `no_relevant_policy`。若存在政策证据，则将本次检索到的 chunk 组织成证据包，交给模型生成 JSON，再进行引用校验。

## 模块职责

- `policy_retrieval_tool`：只负责召回政策证据。
- `policy_qa_service`：编排检索、证据组织、模型生成和引用校验。
- `deepseek_client`：只负责调用 OpenAI 兼容模型。
- `citation_grounding_verifier`：只负责校验模型引用是否来自本次检索证据。
- `policy_qa_tool`：对外提供统一 JSON 返回。

## Prompt 约束

Prompt 要求模型只能依据本次政策证据回答，不能编造未检索到的规则、期限、金额或操作能力。模型必须输出 JSON object，并列出引用的 chunk_id。

## 引用校验机制

引用校验是确定性逻辑，不调用 LLM。它检查引用是否为空、是否重复、是否属于本次 retrieved_chunks。该机制只能说明引用来自本次证据，不能绝对保证完整语义正确。

## 失败处理与有限重试

正常情况下只调用一次模型。如果模型输出非 JSON 或缺少必填字段，允许一次格式修复重试。第二次仍失败时返回 `generation_error`，不进行无限重试。

## 评测指标

自动评测统计成功率、状态匹配率、引用有效率、期望来源命中率、无相关政策识别准确率、人工处理匹配率和平均延迟。

## 人工检查边界

回答是否完整、语气是否合适、是否正确覆盖例外情况，需要人工根据 `manual_checkpoints` 进一步审核。自动指标不能替代人工质量评估。

## 已知限制

当前所有政策均为模拟政策。当前问答只基于检索到的 TopK chunk。当前不会查询订单、不会判断某笔订单退款资格，也不会执行退款、取消、工单创建等真实动作。
