# 重试、错误分类与安全降级

本文档说明当前售后 Agent 的有限重试和降级策略。项目仍是本地模拟电商售后 Agent，不是生产级系统。

## 业务结果、技术异常与可重试错误

业务结果不是系统异常，例如：

- 缺少订单号；
- 订单不存在；
- 退款条件不满足；
- 用户尚未确认创建工单；
- 用户取消；
- 高风险问题转人工；
- 无关请求。

技术异常是系统或外部依赖发生的问题，例如 LLM 超时、网络错误、限流、配置错误或返回结构不合法。

可重试错误仅限只读外部调用中的临时技术异常，例如：

- `llm_timeout`
- `llm_network_error`
- `llm_rate_limit`

配置错误、参数错误、schema 错误、订单不存在、缺订单号等不应重试。

## 为什么创建工单不能自动重试

创建工单是写操作，会改变本地模拟数据库。网络重试或用户重复点击可能导致重复写入，因此本项目使用幂等保护防重复创建，而不是对写操作做自动重试。

幂等保护和失败重试是两件事：

- 幂等：同一写操作重复提交时返回同一结果；
- 重试：临时技术失败后再次执行调用。

当前阶段不对 `order_tool.create_ticket` 或任何 SQLite 写库操作做自动重试。

## 重试策略

默认配置：

- 初始调用 1 次；
- 最多额外重试 2 次；
- 使用有限指数退避；
- 测试可将等待时间配置为 0，避免变慢；
- 不使用随机抖动，保证测试可复现。

环境变量：

```text
READ_ONLY_RETRY_MAX_RETRIES=2
READ_ONLY_RETRY_BASE_DELAY_SECONDS=0.2
READ_ONLY_RETRY_BACKOFF_MULTIPLIER=2.0
```

## 什么是降级

降级是指模型生成不可用时，不编造政策答案，而是返回明确提示和已检索到的政策证据摘要。

例如：已检索到政策 chunk，但 LLM 连续超时，本项目返回：

- `answer_status=degraded`
- `degraded=true`
- `error_category=llm_timeout`
- `fallback_action=return_retrieved_policy_evidence`
- 有限、脱敏的证据摘要和合法引用

## 为什么模型失败时不能编造答案

售后政策涉及退款、保修、优惠券、投诉等业务边界。模型失败后如果继续生成看似确定的答案，容易误导用户。本项目宁可返回降级提示，也不虚构政策结论。

## 当前局限

- 单机 SQLite 模拟环境；
- 无真实用户认证；
- 无分布式任务队列；
- 无真实线上监控告警；
- 未实现复杂熔断、限流或自动恢复；
- 非生产级服务。
